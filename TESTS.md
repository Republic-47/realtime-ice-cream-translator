### Техническое тестирование
Мы спроектировали тесты для проверки ключевых метрик: 
- latency (задержка от захвата до вывода), 
- точность перевода (WER/CER для ASR, BLEU для NMT), 
- стабильность (CPU/GPU load, dropout rate), 
- качество аудио (MOS для синтеза). 

Тестировали на следующем железе: Intel i7 (CPU-only) и NVIDIA RTX 3060 (GPU). Данные: 10-минутные аудио/видео с GitHub (TED talks, podcasts) на английском→русский, китайском→русский, русском→английский. Chunk size: 1-2 сек. Инструменты: PyDub для метрик, Prometheus для мониторинга.

### Результаты технического тестирования

| Метрика            | Тест-кейс          | Значение (CPU) | Значение (GPU) | Цель     | Статус |

**Выводы:** Система держит low-latency на GPU, VAD эффективно фильтрует шум. Бутылочное горлышко — CPU inference (рекомендуем quantization модели для мобильных).

### Тестирование на фокус-группе

**Фокус-группа:** 8 человек (4 из IT, 4 неспециалисты; ages 25-45).

**Сценарии:** просмотр YouTube/Zoom с переводом в реал-тайм (EN/ZH→RU). 

**Задачи:** оценить по шкале 1-5 (синхронизация, естественность, usability). 

**Длительность:** 20 мин/чел + опрос (NPS, open feedback). 

**Инструменты:** Google Forms, screen recording
