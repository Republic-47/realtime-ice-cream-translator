# 🍦 Real-time Ice Cream Translator

Система потокового перевода аудио- и видеоконтента в режиме реального времени с функцией клонирования голоса.

Проект предназначен для захвата звука конкретного системного окна (игры, браузера, плеера), его обработки через локальный микросервисный каскад нейросетей (ASR -> LLM -> TTS) и генерации переведенной речи параллельно оригинальному звуковому ряду с сохранением интонации и тембра спикера.

## ✨ Ключевые особенности (Features)

* 🗣️ **Voice Cloning (Клонирование голоса):** Использование современных TTS-моделей позволяет на лету копировать тембр и эмоции оригинального спикера, создавая эффект естественного дубляжа.
* 🎯 **Targeted Audio Capture & Ducking:** Прямой перехват звука из конкретно выбранного приложения (Windows Audio Loopback). Программа автоматически приглушает звук переводимой игры или видео, не затрагивая системные уведомления, фоновую музыку или голосовой чат (Discord).
* 🧠 **Модульный LLM-каскад:** В отличие от "черных ящиков" End-to-End моделей, перевод осуществляется полноценной LLM (Qwen3). Это позволяет сглаживать артефакты распознавания, понимать контекст диалога и следовать жестким системным инструкциям.
* 🔒 **100% Data Privacy:** Все вычисления происходят локально. Ваши данные не отправляются на серверы сторонних корпораций.

## ⚙️ Схема работы пайплайна

Архитектура разделена на легкий Python-клиент и тяжелый бэкенд в Docker:

1. 🎤 **Audio Input (Клиент):** Перехват звука конкретного окна и фильтрация тишины с помощью алгоритма Silero VAD.
2. 📝 **ASR (Микросервис):** Распознавание накопленной фразы "в один проход" с помощью `Qwen3-ASR`.
3. 🌐 **Machine Translation (Микросервис):** Контекстный перевод текста с помощью `Qwen3-1.7B` (на базе vLLM/Transformers).
4. 🎭 **TTS (Микросервис):** Синтез речи с клонированием голоса через `Faster-Qwen3-TTS`.
5. 🎧 **Audio Output (Клиент):** Воспроизведение сгенерированной аудиодорожки в наушники пользователя.

## 💻 Системные требования

Проект требует значительных вычислительных мощностей из-за одновременной работы нескольких нейросетей:
* 🪟 **ОС:** Windows 10/11 (необходимо для работы `pycaw` и захвата аудиосессий).
* 🚀 **GPU:** Видеокарта NVIDIA с **16+ ГБ VRAM**.
* 💾 **Диск:** ~150 ГБ свободного места (для образов Docker и кэша моделей).
* 🐍 **ПО:** Строго **Python 3.10** (не выше) и установленный **Docker Desktop**.

## 🚀 Установка и запуск

Клиентская часть и Docker-контейнеры могут быть запущены независимо друг от друга в любом порядке. Подключение по вебсокетам произойдет автоматически при инициализации перевода в клиенте.

### 🐳 1. Запуск бэкенда (Docker)
При первом запуске потребуется время на скачивание весов моделей (около 15-20 ГБ).

Сборка образов:
```powershell
docker compose build
```

Запуск контейнеров в фоновом режиме:
```powershell
docker compose up -d
```

Для полной остановки и удаления контейнеров:
```powershell
docker compose down -v
```

### 🖥️ 2. Запуск клиента

Сначала необходимо установить нужные зависимости и пакеты, для этого выполните:

```powershell
python -m venv app/.venv
./app/.venv/Scripts/activate
pip install -r requirements.txt
```

Затем в любой момент просто внутри окружения вызываете:
```powershell
python app/app.py
```

## 🛠️ Возможности модификации стека (Customization)

Благодаря микросервисной архитектуре и общению модулей по WebSockets, пайплайн не привязан намертво к текущим весам. Энтузиасты могут легко модифицировать стек под свое железо или задачи:

* 🧠 **Machine Translation (MT):** Самый гибкий узел. Текущая модель `Qwen3-1.7B` (запущенная через Transformers) может быть заменена на более тяжелые LLM (например, `Qwen3.5-4B` или `Llama-3-8B`), если у вас 24+ ГБ VRAM, для получения идеального литературного перевода. Альтернативно, этот микросервис можно переписать под использование внешних API (OpenAI, Anthropic, DeepL) — это полностью разгрузит видеокарту, оставив на ней только ASR и TTS.
* 📝 **ASR (Распознавание):** Сейчас используется `Qwen3-ASR-1.7B`. Если нужно освободить пару гигабайт видеопамяти, её можно даунгрейднуть до `Qwen3-ASR-0.6B`. Логика `asr_server.py` поддерживает любые модели экосистемы Hugging Face, работающие через стандартный метод `transcribe`.
* 🎭 **TTS (Синтез речи):** Используется `Qwen3-TTS-12Hz-0.6B-Base`. Поскольку клиент отправляет в TTS только текст и base64-аудио для клонирования, бэкенд можно заменить на любой другой движок (например, XTTSv2 или CosyVoice), просто адаптировав приемник JSON-событий в соответствующем контейнере.

## ⚠️ Известные проблемы и ограничения

* ⏱️ **Задержка (Latency):** Из-за физики потокового накопления звука (ожидание конца фразы) и времени инференса трех нейросетей, базовая сквозная задержка перевода составляет около 6–8 секунд.
* 🐢 **Нарастающая задержка (Accumulating Latency):** При переводе с емких языков (английский, китайский) на более "длинные" (например, русский) возникает проблема рассинхронизации. Русский текст физически длиннее на 15–20%. Если оригинальный спикер говорит быстро и без пауз, диктор (TTS) не успевает проговорить переведенную фразу за то же время. Из-за этого очередь аудиофрагментов начинает копиться, и изначальная задержка в 6 секунд может постепенно увеличиться до 10–15 секунд к концу длинного монолога.
* 🪟 **Только Windows:** На данный момент клиент жестко привязан к Windows API для управления звуковыми сессиями.
