# Анализ целевой аудитории

Данный документ содержит актуальное профилирование целевой аудитории open-source проекта **Real-time Ice Cream Translator**. В связи с переходом проекта на микросервисную архитектуру (ASR + MT + TTS на базе Qwen3) с поддержкой **клонирования голоса (Voice Cloning)**, позиционирование продукта и портрет его основного пользователя претерпели значительные изменения.

## 1. Позиционирование проекта

Проект эволюционировал из простого локального переводчика в **продвинутую станцию перевода реального времени с сохранением интонаций и тембра спикера**.
Ключевые ценности (Киллер-фичи):
* **Voice Cloning & Time-Stretch:** Сохранение оригинальных эмоций спикера и синхронизация длины фраз.
* **Точечный захват звука:** Перехват и приглушение аудио конкретного приложения, не затрагивая системные звуки и голосовые чаты (Discord).
* **100% Data Privacy:** Локальный инференс без утечек аудиопотока на корпоративные серверы.
* **Модульность:** Возможность кастомизации пайплайна (замена локальных моделей, подключение внешних API).

## 2. Технические требования для пользования продуктом

Архитектура локальных нейросетей диктует жесткие ограничения, определяющие ядро текущей аудитории (премиум-сегмент железа и энтузиасты):
* **Аппаратные требования (Hard Requirements):** Наличие видеокарты с **~16 ГБ VRAM** (уровня RTX 4080 / 3090 / 4070 Ti Super) для одновременного инференса трех нейросетей.
* **Платформа:** На данный момент, строго **Windows** с поддержкой **WSL** (обусловлено использованием Windows Audio Session API / `pycaw` для точечного управления звуком).
* **Техническая грамотность:** На текущем этапе пользователи должны обладать базовыми навыками работы с терминалом и Docker для развертывания бэкенда.

## 3. Сегменты аудитории

#### Сегмент А: AI-Энтузиасты и Open-Source комьюнити
* **Контекст использования:** Разворачивание локальных ИИ-пайплайнов, эксперименты с новыми моделями.
* **Болевые точки:** Закрытые экосистемы (проприетарный софт), невозможность "залезть под капот" и настроить логику работы.
* **Ключевая потребность:** Модульность. Возможность легко заменить тяжелую LLM-модель на другую, либо переключиться на внешний API (DeepL, OpenAI), если локальных ресурсов не хватает.

#### Сегмент B: Контент-мейкеры, Стримеры и фанаты VTuber'ов
* **Контекст использования:** Просмотр и рестриминг зарубежного контента (китайский/японский сегмент, аниме, коллаборации).
* **Болевые точки:** Классический одноголосый машинный перевод убивает эмоции и атмосферу трансляции.
* **Ключевая потребность:** Клонирование голоса (Voice Cloning) для эффекта "естественного дубляжа" и возможность вынести тяжелый бэкенд на отдельный стримерский ПК (Dual-PC setup).

#### Сегмент C: Хардкорные геймеры (Азиатские MMO и Visual Novels)
* **Контекст использования:** Прохождение нелокализованных игр (гачи, JRPG, инди-релизы).
* **Болевые точки:** Экранные переводчики (OCR) заставляют читать текст, отвлекая от геймплея. Браузерные расширения не работают в полноэкранных 3D-приложениях.
* **Ключевая потребность:** Фоновый системный захват конкретного окна игры без приглушения фоновой музыки (Spotify) или голосов тиммейтов.

#### Сегмент D: Иностранные студенты и экспаты
* **Контекст использования:** Обучение в зарубежных вузах (например, студенты из Африки или Китая в России), просмотр лекций и вебинаров.
* **Болевые точки:** Сложность восприятия быстрой академической речи, отставание субтитров.
* **Ключевая потребность:** Высокая точность перевода терминологии (при использовании готовых API) и минимальная задержка аудио.

#### Люди, работающие с конфиденциальной информацией
* **OSINT-специалисты и аналитики:** Перехват и анализ зарубежных новостных сводок и радиоэфиров. Строгая необходимость в оффлайн-работе и защите данных.
* **Профессионалы на удаленке (NDA):** Участие в корпоративных звонках на иностранных языках, где политики безопасности запрещают использование облачных ИИ-решений.

## 4. Пользовательские альтернативы

Как пользователи решают свои проблемы прямо сейчас (до использования нашего продукта):
* **Браузерные расширения (Яндекс Переводчик видео):** Работают только в поддерживаемых браузерах, ломаются при обновлениях сайтов, не применимы для игр и десктопного софта.
* **Google Lens / Экранные OCR-переводчики:** Требуют постоянного внимания (чтение текста вместо погружения в контент), ломают динамику видео или игры.
* **Субтитры реального времени (Windows Live Captions / Chrome):** Часто не поддерживают нужную языковую пару (например, прямой перевод с китайского на русский), не передают эмоции.

## 5. Влияние на будущую разработку

Анализ аудитории напрямую диктует следующие приоритеты для Roadmap:

1. **Развитие Client-Server архитектуры (Фокус на Сегмент B и C):** Упаковка легкого Python-клиента с графическим интерфейсом (GUI) в `.exe` инсталлятор. Бэкенд (Docker) должен иметь возможность работать автономно на удаленном сервере (по SSH/IP).
2. **Оптимизация VRAM (Демократизация проекта):** Для расширения базы пользователей необходимо внедрить продвинутые методы квантования и поддержку GGUF/llama.cpp форматов. Стратегическая цель — снижение минимального порога с 16 ГБ до **8-12 ГБ VRAM** для расширения целевой аудитории.
3. **Модульность в UI (Фокус на Сегмент А):** Добавление в графический интерфейс селектора "Режим работы" (Локальный LLM / Внешний API), позволяющего использовать облачные мощности для этапа машинного перевода (MT).
4. **Кроссплатформенность (Long-term):** После стабилизации функционала на Windows, исследование альтернативных библиотек аудио-захвата для портирования клиента на Linux и macOS.
